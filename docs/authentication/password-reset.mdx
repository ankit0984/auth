# Password Reset

## Request reset email

```http
POST /api/auth/forgot-password
Content-Type: application/json
```

### Request body

```json
{
  "email": "john@example.com"
}
```

### Behavior

- Finds the user by email
- Creates a secure random reset token
- Signs that token into a time-limited JWT
- Stores the raw reset token and expiry in MongoDB
- Sends the reset link by email

## Complete reset

```http
POST /api/auth/reset-password
Content-Type: application/json
```

### Request body

```json
{
  "token": "reset-jwt-from-email-link",
  "newPassword": "NewSecurePass123!"
}
```

### Behavior

- Validates the signed reset token
- Confirms the stored token is still active
- Updates the password
- Clears the reset token fields
- Clears the refresh token
- Clears any pending 2FA challenge

## Security implications

- Old sessions are invalidated after password reset
- Expired reset tokens cannot be reused
- Password policy is enforced during reset

## Common errors

- `404 user not found`
- `400 token and new password are required`
- `400 invalid or expired reset token`
- `400 password must be 8-64 chars and include uppercase, lowercase, number and special character`
